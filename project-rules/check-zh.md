# 检查

执行全面的代码质量和安全检查。

## 主要任务:
运行`npm run check`(或项目特定的检查命令)并解决任何产生的错误。

## 重要提醒:
- 在此过程中不要提交任何代码
- 不要更改版本号
- 仅专注于修复检查发现的问题

## 常见检查包括:
1. **代码检查**: 代码风格和语法错误
2. **类型检查**: TypeScript/Flow类型错误
3. **单元测试**: 失败的测试用例
4. **安全扫描**: 漏洞检测
5. **代码格式化**: 风格一致性
6. **构建验证**: 编译错误

## 流程:
1. 运行检查命令
2. 分析输出中的错误和警告
3. 按优先级顺序修复问题:
   - 首先修复中断构建的错误
   - 测试失败
   - 代码检查错误
   - 警告
4. 每次修复后重新运行检查
5. 继续直到所有检查通过

## 对于不同项目类型:
- **JavaScript/TypeScript**: `npm run check` 或 `yarn check`
- **Python**: `black`, `isort`, `flake8`, `mypy`
- **Rust**: `cargo check`, `cargo clippy`
- **Go**: `go vet`, `golint`
- **Swift**: `swift-format`, `swiftlint`

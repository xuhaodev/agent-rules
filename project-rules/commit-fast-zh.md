# 快速提交任务

基于暂存的变更生成3个提交消息建议，然后自动使用第一个建议而无需用户确认。

遵循规范提交格式，使用适当的表情符号并创建解释变更目的的描述性消息。跳过手动消息选择步骤以简化提交过程。

## 步骤:
1. 运行`git status`查看暂存的变更
2. 遵循规范提交格式生成3个提交消息建议
3. 自动选择第一个建议
4. 使用选定的消息执行`git commit -m`
5. 从提交中排除Claude共同作者页脚

## 提交类型:
- ✨ feat: 新功能
- 🐛 fix: 错误修复
- 📝 docs: 文档变更
- ♻️ refactor: 代码重构
- 🧑‍💻 chore: 工具和维护
- 🎨 style: 代码格式化、缺失分号等
- ⚡️ perf: 性能改进
- ✅ test: 添加或纠正测试
